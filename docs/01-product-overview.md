# 01 · 产品总览与用户画像

## 1.1 产品定位

**一句话**：把任何长文，10 分钟内变成可发布的短视频矩阵。

**三类核心价值**：

1. **降本**：替代"编导 + 文案 + 剪辑 + 配音"4 个岗位 60% 的重复劳动。
2. **提效**：单条内容产出从 4~8 小时缩短到 5~15 分钟。
3. **放量**：同一长文一键生成 N 个平台版本（抖音 / 小红书 / 视频号 / B 站 / YouTube Shorts），并支持账号矩阵分发。

## 1.2 目标用户与典型场景

### 用户画像 A：个人 / 兼职创作者
- 痛点：选题难、剪辑慢、坚持不了。
- 场景：把每天读到的好文章，自动转成 30~60 秒口播短视频。
- 关心：模板丰富度、TTS 是否自然、平台账号能否托管。

### 用户画像 B：MCN / 工作室运营
- 痛点：账号矩阵成本高、人均产能低。
- 场景：1 个运营管 50 个账号，平台批量生成"千号千面"内容。
- 关心：批量任务、风格隔离、防关联风控、数据看板。

### 用户画像 C：品牌 / 企业新媒体
- 痛点：白皮书 / 产品文档不会变成种草内容。
- 场景：上传产品 PDF → 生成 20 条小红书种草笔记 + 5 条抖音口播。
- 关心：品牌词库、敏感词审核、风格一致性、合规。

### 用户画像 D：知识付费 / 教育创作者
- 痛点：长课不能直接当抖音引流。
- 场景：把 1 小时课程切 30 条 1 分钟短视频钩子。
- 关心：知识点精准度、字幕准确率、引流转化。

## 1.3 核心用户旅程（User Journey）

```
注册登录
   │
   ▼
绑定平台账号（抖音/小红书/视频号/B站/YouTube）
   │
   ▼
导入素材：URL / PDF / Word / Markdown / RSS / 知识库
   │
   ▼
选择"风格预设"（口播 / 图文 / 信息流 / 影视感 / 卡通 ...）
   │
   ▼
一键生成：标题 + 文案 + 分镜 + 图片 + TTS + 视频
   │
   ▼
预览 / 二次编辑（Web 端可视化时间线）
   │
   ▼
选择账号 + 平台 + 定时
   │
   ▼
一键发布 / 定时发布
   │
   ▼
数据回流：播放、点赞、评论、引流转化
```

## 1.5 非功能性需求

- **性能**：单条短视频端到端 ≤ 5 分钟（不含人工编辑）。
- **可用性**：核心 API 可用率 ≥ 99.5%，发布失败自动重试。
- **可扩展性**：模型层可插拔（OpenAI / Claude / Gemini / 通义 / 文心 / Kling / Sora / 可灵 / 即梦 ...）。
- **合规**：内容审核（涉政 / 涉黄 / 版权）、水印 / AIGC 标识、平台 ToS 适配。
- **数据安全**：用户素材加密存储、模型调用日志可审计。
- **国际化**：中 / 英 / 日 / 泰 起步，UI 文案与 prompt 模板都支持多语言。

## 1.6 商业化设计（简版）

- **免费版**：每月 N 条短视频额度、低分辨率、带平台水印。
- **创作者版**：去水印、HD、更多模型、3 个平台账号。
- **团队版**：账号矩阵、协作、数据看板、API 调用。
- **企业版**：私有部署、品牌词库、SLA、专属模型微调。

按量计费补充：图片张数、视频秒数、TTS 字符数、LLM Token。
